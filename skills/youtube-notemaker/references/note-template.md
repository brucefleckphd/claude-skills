# Note Template

Use this exact markdown structure for all YouTube notes.

## Template

```markdown
---
tags:
  - 
type: ref.note
source: YouTube
people: author
date-created: {{date:YYYY-MM-DD}}
---
# {Author} - {Title of Video}

**URL:** {video_url}

## Situation
[A description of the history and present conditions that lead to the current problem. Provide context the speaker establishes for why this topic matters now.]

## Problem
[The current problem or challenge that needs to be addressed. What pain point or gap is the speaker identifying?]

## Solution
[The premise proposed by the speaker that offers the best chance of solving the problem with the least time, money, and effort. The core framework or approach being taught.]

## Teaching Points

1. **{Teaching Point 1 Title}:** [Explain the topic/concept]
   - **How to Use:** [Describe how to apply this concept]
   - **Example:** [Provide a concrete example from the video]

2. **{Teaching Point 2 Title}:** [Explain the topic/concept]
   - **How to Use:** [Describe how to apply this concept]
   - **Example:** [Provide a concrete example from the video]

3. **{Teaching Point 3 Title}:** [Explain the topic/concept]
   - **How to Use:** [Describe how to apply this concept]
   - **Example:** [Provide a concrete example from the video]

[Continue adding teaching points as needed - minimum 5 for substantive videos]

## Action Steps

1. {First action step the viewer should take}
2. {Second action step}
3. {Third action step}
[Add as many action steps as the content warrants]


```

## Guidelines

**Minimum Requirements:**
- At least 5 teaching points for videos over 10 minutes
- At least 3 action steps
- All sections must be completed

**Formatting Rules:**
- Bold the first few words of each teaching point
- Use H2 (`##`) for main sections
- Use H6 (`######`) only for the URL footer
- Include the URL in both the header section and footer

**Content Quality:**
- Extract specific, actionable insights - not vague summaries
- Include concrete examples from the video when available
- Action steps should be immediately implementable
- Situation/Problem/Solution should form a cohesive narrative

**Filename Convention:**
`{Author}-{Title}.md`

Example: `Alex Hormozi-How to Build a $100M Business.md`
