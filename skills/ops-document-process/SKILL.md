# Document a Process

Turn tribal knowledge into repeatable systems. This skill creates clear SOPs that anyone can follow.

## When to Use This Skill

- Systematizing recurring tasks
- Onboarding new team members
- Quality control documentation
- Delegation preparation
- Process improvement

## Required Inputs

- Process name and purpose
- Current steps (even if informal)
- Who performs this process
- Frequency and triggers
- Tools/systems involved

## Output Format

```markdown
# Standard Operating Procedure: [Process Name]

**Version:** 1.0
**Last Updated:** [Date]
**Owner:** [Role/Person]
**Review Frequency:** [Quarterly/Annually]

---

## Purpose

[Why this process exists and what it achieves]

---

## When to Use This SOP

**Triggers:**
- [When this process starts]
- [Events that initiate this]

**Frequency:** [How often performed]

---

## Prerequisites

**Access Required:**
- [System/Tool 1]
- [System/Tool 2]

**Materials Needed:**
- [Resource 1]
- [Resource 2]

**Time Required:** [Estimated duration]

---

## Step-by-Step Procedure

### Step 1: [Action]
**Time:** [X minutes]

[Detailed instruction for this step]

> **Tip:** [Helpful hint or common mistake to avoid]

### Step 2: [Action]
**Time:** [X minutes]

[Detailed instruction]

**Sub-steps:**
1. [Sub-step a]
2. [Sub-step b]
3. [Sub-step c]

### Step 3: [Action]
**Time:** [X minutes]

[Detailed instruction]

**Decision Point:**
- If [condition], proceed to Step 4
- If [condition], skip to Step 5

[Continue for all steps...]
```
