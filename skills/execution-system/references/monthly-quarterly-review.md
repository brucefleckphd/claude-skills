# Monthly & Quarterly Review

Assess progress against priority domains, generate Cook Medical project summaries for quarterly performance reviews, and plan upcoming periods.

## Invocation

### Monthly Review
```
Run monthly review for: [MONTH YEAR]
```

### Quarterly Review
```
Run quarterly review for: [Q1/Q2/Q3/Q4 YEAR]
```

## Monthly Review Process

### Step 1: Gather Data

**Read the following files:**
- All daily notes for the month
- Weekly planning notes (if created)
- Monthly to-do list
- Systems Planning document (for domain targets)

## Constraints

### NEVER DO:
- Fabricate data or metrics not found in daily notes
- Skip Cook Medical project summary (required for performance review)
- Ignore gaps or challenges -- be honest in assessment

### ALWAYS DO:
- Pull actual data from daily notes
- Include specific project contributions for Cook Medical
- Assess each priority domain against its target
- Provide actionable focus areas for next period
- Note system improvements that could help
