# Evaldo Decision Trees

Framework selection guidance for different situations.

## Primary Decision Tree: What Are You Writing?

```
STARTING A NEW PROJECT?
│
├─ Complete VSL
│   └─ Use: evaldo (orchestrator)
│      Flow: philosophy → ideas → headlines → leads → body → bullets → close
│
├─ Just Headlines
│   └─ Use: evaldo-headlines
│      Start with: headlines-13-boosters
│      Then: headlines-open-loops
│
├─ Just the Opening
│   └─ Use: evaldo-leads
│      Start with: leads-punch-in-gut
│      Then: leads-first-5-minutes
│
├─ Just the Pitch/Close
│   └─ Use: evaldo-close
│      Start with: close-stacking
│      Then: close-price-anchor → close-guarantee → close-urgency
│
├─ Email Creative/Lifts
│   └─ Use: evaldo-lifts
│      Start with: lifts-dick-method
│      Then: lifts-safe-vs-intriguing
│
└─ Critiquing Copy
    └─ Use: evaldo-critic (agent)
       Or apply manually from evaldo-philosophy
```

## Idea Validation Tree

```
DO I HAVE A GOOD IDEA?
│
├─ Does it pass Three Box?
│   ├─ Emotional? → YES/NO
│   ├─ Simple? → YES/NO
│   └─ New? → YES/NO
│
│   ALL THREE YES? → Proceed to Croc Brain
│   ANY NO? → Return to evaldo-ideas
│
├─ Does it pass Croc Brain?
│   ├─ Lion/Rabbit trigger? → YES/NO
│   ├─ Geico simple? → YES/NO
│   └─ Curiosity new? → YES/NO
│
│   ALL THREE YES? → Proceed to writing
│   ANY NO? → Refine with evaldo-ideas
│
└─ Can you write the One Belief?
    └─ "This new opportunity [___] is the key to [desire]
        and it's only attainable through my [mechanism]"

    CLEAR FILL-INS? → Ready to write
    FUZZY? → More idea work needed
```

## Proof Type Selection Tree

```
WHAT TYPE OF PROOF DO I NEED?
│
├─ Claim: "This works"
│   └─ Best proof: Case studies + Testimonials
│
├─ Claim: "This is safe"
│   └─ Best proof: Expert endorsement + Data
│
├─ Claim: "This is easy"
│   └─ Best proof: Demonstration + Testimonials
│
├─ Claim: "This is new/different"
│   └─ Best proof: Research + Expert quotes
│
└─ Claim: "This is worth the price"
    └─ Best proof: ROI calculations + Comparisons
```

## Story Type Selection Tree

```
WHICH STORY TYPE SHOULD I USE?
│
├─ Do you have real credentials?
│   └─ YES → Consider GURU story
│       "After 30 years in [field]..."
│
├─ Were you once like the prospect?
│   └─ YES → Consider LITTLE GUY story
│       "I was in your exact situation..."
│
├─ Is there a shared frustration/enemy?
│   └─ YES → Consider COMMON ENEMY story
│       "They've been lying to you..."
│
└─ HYBRID OPTION:
    Little Guy who discovered → Common Enemy → Now has Guru knowledge
```

## Close Structure Tree

```
BUILDING THE CLOSE:
│
├─ Step 1: REVEAL THE OFFER
│   └─ Use: close-stacking
│       List all components with values
│
├─ Step 2: ANCHOR THE PRICE
│   └─ Use: close-price-anchor
│       Compare to expensive alternatives
│
├─ Step 3: DROP THE PRICE
│   └─ High anchor → Mid anchor → Actual price
│
├─ Step 4: ADD BONUSES
│   └─ Use: close-bonuses
│       Strategic bonuses that overcome objections
│
├─ Step 5: PRESENT GUARANTEE
│   └─ Use: close-guarantee
│       Make it bold and confident
│
├─ Step 6: CREATE URGENCY
│   └─ Use: close-urgency
│       Real deadline or limitation
│
├─ Step 7: CALL TO ACTION
│   └─ Use: close-cta
│       Clear, simple, benefit-focused
│
└─ Step 8: RE-CLOSE
    └─ Use: close-re-close
        Different angle for fence-sitters
```

## Lift Strategy Tree

```
WHICH LIFT APPROACH?
│
├─ Is list warm/engaged?
│   └─ YES → Use SAFE approach
│       "This video about [topic]..."
│
├─ Is list cold/new?
│   └─ YES → Use INTRIGUING approach
│       "Something strange happened..."
│
├─ Need maximum volume?
│   └─ Use INTRIGUING approach
│       Higher CTR, wider net
│
├─ Need quality over quantity?
│   └─ Use SAFE approach
│       Pre-qualifies viewers
│
└─ NOT SURE?
    └─ Test both approaches
        Compare: CTR × Conversion = Overall
```
