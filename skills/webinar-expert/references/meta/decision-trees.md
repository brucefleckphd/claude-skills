# Webinar Expert Decision Trees

## Purpose

This document provides decision tree frameworks for choosing which expert's approach to use based on context variables. Use these when experts conflict or when you need guidance on which methodology best fits your situation.

## Master Decision Tree: Which Expert's Approach?

```
START: I need to create/optimize a webinar
│
├─► What's the price point?
│   │
│   ├─► Under $500
│   │   └─► BIAS: Fladlien (60-75 min, direct, fast)
│   │       Next: Check market sophistication
│   │
│   ├─► $500-$2,000
│   │   └─► BIAS: Balanced (75-90 min, synthesis)
│   │       Next: Check product complexity
│   │
│   ├─► $2,000-$10,000
│   │   └─► BIAS: Cage (90-120 min, depth, comprehensive)
│   │       Next: Check market sophistication
│   │
│   └─► $10,000+
│       └─► BIAS: Cage Extended (120-180 min, multi-touch)
│           Next: Consider consultation model instead
│
├─► What's the market sophistication? (Eugene Schwartz scale)
│   │
│   ├─► Level 1-2 (Unaware or Problem-Aware)
│   │   └─► BIAS: Fladlien (simple, direct, emotional)
│   │       Frameworks: 150 Yeses, Emotional Levers, Simple transitions
│   │
│   ├─► Level 3 (Solution-Aware)
│   │   └─► BIAS: Balanced (prove why you, not just why anything)
│   │       Frameworks: Mix of both, context-dependent
│   │
│   └─► Level 4-5 (Product-Aware or Most Aware)
│       └─► BIAS: Cage (nuanced, sophisticated, desire tap)
│           Frameworks: 8-Question X-Ray, Desire Tap, Deep content
│
├─► What's the webinar type?
│   │
│   ├─► Live (one-time or recurring live)
│   │   └─► BIAS: Cage retention + Fladlien close
│   │       Why: Need community feeling + conversion
│   │       Q&A: 20-30 minutes (Cage approach)
│   │
│   ├─► Automated Evergreen
│   │   └─► BIAS: Fladlien structure + Cage content
│   │       Why: Efficiency critical + pre-handle objections
│   │       Q&A: Recorded FAQ, 10-15 min (Fladlien approach)
│   │
│   └─► Hybrid (Live + Replay)
│       └─► BIAS: Balanced (optimize for both)
│           Why: Live experience + replay conversion
│           Q&A: 15-20 minutes (split difference)
│
├─► What's the primary goal?
│   │
│   ├─► Maximize immediate conversion
│   │   └─► BIAS: Fladlien (commitment psychology, tight close)
│   │       Frameworks: 150 Yeses, Price Cascade, Stack Slide
│   │
│   ├─► Build long-term relationship
│   │   └─► BIAS: Cage (depth, value, community)
│   │       Frameworks: Content Chunks, Stick Strategy, Comprehensive Q&A
│   │
│   └─► Both (conversion + relationship)
│       └─► BIAS: Synthesis (best of both)
│           Structure: Cage open/middle, Fladlien close
│
└─► What's the product type?
    │
    ├─► Information/Education (Course, Training)
    │   └─► BIAS: Cage (depth proves competence)
    │       Structure: Comprehensive chunks, teach-apply-transition
    │
    ├─► Transformation (Coaching, Personal Development)
    │   └─► BIAS: Fladlien (emotional leverage critical)
    │       Structure: Emotional Levers, transformation promise
    │
    ├─► Service (Done-for-you, Agency)
    │   └─► BIAS: Balanced (prove competence + urgency)
    │       Structure: Cage proof + Fladlien close
    │
    └─► Physical Product
        └─► BIAS: Fladlien (direct, benefits-focused)
            Structure: Simple, fast, demonstration-heavy
```

## Decision Tree: Opening Sequence Design

```
Opening Sequence Question: How should I start this webinar?

START: What does the audience know about me?
│
├─► Unknown speaker (cold audience)
│   ├─► Authority Crisis: Need credibility FAST
│   │   Strategy: Proof-heavy open (Fladlien)
│   │   Elements:
│   │   • Results in first 2 minutes
│   │   • Third-party validation
│   │   • Credentials/social proof
│   │   • Then: Desire Tap (Cage) to connect
│   │   Duration: 5-7 minutes before content
│   │
│   └─► Check: What's the sophistication?
│       │
│       ├─► Low sophistication
│       │   └─► Simple authority + emotional hook (Fladlien)
│       │       "I've helped 10,000 people do X. Today you'll learn..."
│       │
│       └─► High sophistication
│           └─► Authority + contrarian insight (Cage)
│               "You've probably tried X. Here's why it didn't work..."
│
├─► Known speaker (warm audience)
│   ├─► Authority established
│   │   Strategy: Jump faster to desire (Cage)
│   │   Elements:
│   │   • Brief reminder of credibility (1-2 min)
│   │   • Desire Tap (activate what they want)
│   │   • Transformation Promise
│   │   • Begin content faster
│   │   Duration: 3-5 minutes before content
│   │
│   └─► Check: What's the relationship?
│       │
│       ├─► First webinar with this audience
│       │   └─► Medium authority build (3-4 min)
│       │
│       └─► Repeat webinar/existing customers
│           └─► Minimal authority (1-2 min), jump to value
│
└─► Mixed audience (some know, some don't)
    ├─► Default: Balanced approach
    │   Strategy: Quick authority + desire tap
    │   Elements:
    │   • 2-3 min credibility (helps newcomers)
    │   • Acknowledge existing relationship (for warm audience)
    │   • Desire Tap (works for both)
    │   • Transformation Promise (universal)
    │   Duration: 5 minutes before content
    │
    └─► Optimize for: Newcomers or existing?
        │
        ├─► Optimize for newcomers (building list)
        │   └─► More authority (Fladlien), then Cage desire
        │
        └─► Optimize for existing (monetization)
            └─► Less authority, more value (Cage bias)
```

## Decision Tree: Content Structure

```
Content Question: How should I structure the teaching portion?

START: What's the webinar length?
│
├─► 60-75 minutes (short format)
│   ├─► Content time available: ~30-40 minutes
│   │   Strategy: Fladlien-style focus
│   │   Structure:
│   │   • 3 major teaching points MAX
│   │   • Simple, clear, actionable
│   │   • Less depth, more breadth
│   │   • Quick transitions (4-part system)
│   │   Chunks: 3 × 10-12 minutes each
│   │
│   └─► Check: Complexity of topic
│       │
│       ├─► Simple/single concept
│       │   └─► 3 chunks: What, Why, How (conceptual)
│       │       Apply: Fladlien simplicity
│       │
│       └─► Complex/multi-concept
│           └─► Problem: Not enough time
│               Solution: Multi-webinar series or go longer
│
├─► 75-90 minutes (medium format)
│   ├─► Content time available: ~40-50 minutes
│   │   Strategy: Balanced approach
│   │   Structure:
│   │   • 3-4 major chunks
│   │   • Moderate depth
│   │   • Cage's Teach-Apply-Transition per chunk
│   │   • Fladlien's transitions between chunks
│   │   Chunks: 3-4 × 12-15 minutes each
│   │
│   └─► Check: Audience sophistication
│       │
│       ├─► Lower sophistication
│       │   └─► 3 chunks, more application time
│       │       Focus: Understanding over comprehensiveness
│       │
│       └─► Higher sophistication
│           └─► 4 chunks, more nuance
│               Focus: Depth over simplicity
│
├─► 90-120 minutes (long format)
│   ├─► Content time available: ~50-70 minutes
│   │   Strategy: Cage-style depth
│   │   Structure:
│   │   • 4-5 comprehensive chunks
│   │   • Deep teaching + application
│   │   • Full Teach-Apply-Transition (Cage)
│   │   • Stick Strategy between chunks (prevent drop-off)
│   │   Chunks: 4-5 × 12-18 minutes each
│   │
│   └─► Check: Product complexity
│       │
│       ├─► High complexity (system, methodology)
│       │   └─► 5 chunks, comprehensive coverage
│       │       Each chunk: Complete sub-system
│       │
│       └─► Medium complexity
│           └─► 4 chunks, focused depth
│               Each chunk: Major component
│
└─► 120+ minutes (extended format)
    ├─► Content time available: ~70-90 minutes
    │   Strategy: Cage comprehensive
    │   Structure:
    │   • 5-7 chunks
    │   • Workshop-style depth
    │   • Include exercises/worksheets
    │   • Strategic breaks (Stick Strategy critical)
    │   Chunks: 5-7 × 12-18 minutes + exercises
    │
    └─► Warning: Retention risk HIGH
        Solutions:
        • Multiple Stick Strategies deployed
        • Break every 45 minutes
        • Bonus reveals throughout
        • Interactive elements
        • Consider: Is this better as multi-part series?
```

## Decision Tree: Close Architecture

```
Close Question: How should I structure the offer presentation?

START: What's the offer complexity?
│
├─► Simple offer (1-3 components)
│   ├─► Strategy: Fladlien direct
│   │   Structure:
│   │   • Clear statement of offer
│   │   • Brief component breakdown
│   │   • Price reveal (possibly early)
│   │   • Simple urgency
│   │   Duration: 10-15 minutes
│   │
│   └─► Check: Price point
│       │
│       ├─► Low price (under $500)
│       │   └─► Fast close, minimal justification needed
│       │       Elements: Offer → Price → Urgency → CTA
│       │
│       └─► Higher price ($500+)
│           └─► Need more justification despite simplicity
│               Add: Price Cascade (Fladlien) for anchoring
│
├─► Medium complexity (4-7 components)
│   ├─► Strategy: Fladlien Stack Slide
│   │   Structure:
│   │   • Sequential component reveal
│   │   • Value assigned to each
│   │   • Running total builds
│   │   • Price Cascade
│   │   • Bonus reveal (2-3x time on bonuses)
│   │   • Final urgency
│   │   Duration: 15-25 minutes
│   │
│   └─► Check: Audience familiarity
│       │
│       ├─► Familiar with this type of offer
│       │   └─► Can move faster through stack
│       │       Focus: Differentiation, not education
│       │
│       └─► Unfamiliar offer type
│           └─► Need to explain each component more
│               Add: Case study per major component
│
├─► High complexity (8+ components, bundles, tiers)
│   ├─► Strategy: Cage comprehensive + Fladlien stack
│   │   Structure:
│   │   • Group components logically
│   │   • Explain transformation per group
│   │   • Stack Slide for visual accumulation
│   │   • Bonuses (2-3x time rule)
│   │   • Objection sweep before price
│   │   • Price Cascade
│   │   • Detailed what-happens-next
│   │   Duration: 25-35 minutes
│   │
│   └─► Check: Risk of confusion?
│       │
│       ├─► High confusion risk
│       │   └─► Simplify grouping, clear summary slide
│       │       Add: "Here's everything you get" recap
│       │
│       └─► Low confusion risk
│           └─► Can show full complexity
│               Benefit: Demonstrates value depth
│
└─► Multiple tiers/options
    ├─► Strategy: Depends on goal
    │   │
    │   ├─► Goal: Push premium tier
    │   │   └─► Present ascending order (good → better → best)
    │   │       Psychology: Anchoring to highest price
    │   │       Close: Fladlien urgency on premium tier
    │   │
    │   └─► Goal: Maximize total buyers
    │       └─► Present descending order (best → better → good)
    │           Psychology: Preference for seen-first option
    │           Close: Offer choice, slight bias to middle
    │
    └─► Warning: Multiple options reduce conversion
        Solution: Strong recommendation for one tier
        "Most people choose X because..."
```

## Decision Tree: Objection Handling Strategy

```
Objection Question: How should I handle objections?

START: When do I know what the objections are?
│
├─► Before webinar (market research done)
│   ├─► Strategy: Cage pre-handling
│   │   Method:
│   │   • Map all 10 Universal Objections
│   │   • Add market-specific objections (from 8-Question X-Ray)
│   │   • Handle during content (not after)
│   │   • Weave into teaching naturally
│   │   Result: Clean close, minimal Q&A objections
│   │
│   └─► Check: How many objections identified?
│       │
│       ├─► 10-15 objections
│       │   └─► Standard approach: Address during content
│       │       Map: 2-3 objections per content chunk
│       │
│       └─► 20+ objections
│           └─► High-objection market (skeptical, burned before)
│               Strategy: Dedicate objection-handling section
│               "Let me address what you're probably thinking..."
│
├─► During webinar (live objections appearing)
│   ├─► Strategy: Fladlien reversal system
│   │   Method:
│   │   • Listen for objection
│   │   • Acknowledge validity
│   │   • Reverse to reason to buy
│   │   • "That's exactly why you need this"
│   │   Context: Live Q&A sessions
│   │
│   └─► Check: Objection type
│       │
│       ├─► Stall objection ("Need to think about it")
│       │   └─► Response: Remove thinking barrier
│       │       "What specifically do you need to think about?"
│       │       Address that specific concern
│       │
│       ├─► Real objection (genuine concern)
│       │   └─► Response: Proof + guarantee
│       │       Show how others with same concern succeeded
│       │
│       └─► Smokescreen objection (hiding real issue)
│           └─► Response: Dig deeper
│               "Is it really X, or is it something else?"
│
└─► Automated webinar (can't handle live)
    ├─► Strategy: Comprehensive pre-handling (Cage)
    │   Method:
    │   • Anticipate ALL objections
    │   • Address in content
    │   • Record FAQ section
    │   • Post-webinar email sequence for lingering objections
    │   Critical: Can't course-correct, must be complete
    │
    └─► Check: Testing results
        │
        ├─► Specific objection appearing repeatedly
        │   └─► Solution: Add dedicated section addressing it
        │       Place: Before close, after content
        │
        └─► General resistance, not specific objection
            └─► Solution: Increase proof throughout
                Add: More testimonials, case studies, data
```

## Decision Tree: Title Development

```
Title Question: How do I create the webinar title?

START: What's the market awareness level?
│
├─► Level 1-2 (Unaware or Problem-Aware)
│   ├─► Strategy: Problem-focused title (Fladlien simplicity)
│   │   Formula: "How to [Solve Problem] in [Timeframe]"
│   │   Example: "How to Double Your Sales in 90 Days"
│   │   Test: Does it speak to felt pain?
│   │
│   └─► Apply: Cage's Title Alchemy
│       │
│       ├─► Magnetic: "Double Your Sales"
│       ├─► Specific: "in 90 Days"
│       ├─► Urgent: "Before Year-End"
│       └─► Curiosity: "Using the 3-Touch Method"
│       Result: "How to Double Your Sales in 90 Days Before Year-End Using the 3-Touch Method"
│
├─► Level 3 (Solution-Aware)
│   ├─► Strategy: Better-solution title (Cage nuance)
│   │   Formula: "The [New/Better] Way to [Achieve Desire]"
│   │   Example: "The Faster Way to Build Your Email List"
│   │   Test: Does it promise superiority over known solutions?
│   │
│   └─► Apply: Title Alchemy + differentiation
│       Elements:
│       • Magnetic: Promise of better result
│       • Specific: How it's different
│       • Urgent: Why now matters
│       • Curiosity: Unique mechanism
│
├─► Level 4-5 (Product-Aware or Most Aware)
│   ├─► Strategy: Contrarian or mechanism title (Cage sophistication)
│   │   Formula: "[Counterintuitive Truth] for [Specific Outcome]"
│   │   Example: "Why Posting More Often Kills Your Engagement (And What to Do Instead)"
│   │   Test: Does it challenge conventional wisdom?
│   │
│   └─► Apply: Title Alchemy + controversy
│       Elements:
│       • Magnetic: Contrarian hook
│       • Specific: Exact outcome promised
│       • Urgent: Implied (doing wrong thing now)
│       • Curiosity: The alternative approach
│
└─► Unknown market awareness
    ├─► Strategy: Research first (Cage 8-Question X-Ray)
    │   Process:
    │   1. Mine forums/groups for language
    │   2. Identify awareness level from language used
    │   3. Return to appropriate branch above
    │   4. Use market's language in title
    │
    └─► Fallback: Test multiple titles
        Method:
        • Create 3-5 title variations
        • Test with small audience sample
        • Measure registration rate
        • Use winner for main launch
```

## Decision Tree: Q&A Session Design

```
Q&A Question: How should I structure the Q&A?

START: Is this live or automated?
│
├─► Live webinar
│   ├─► Check: What's the primary goal?
│   │   │
│   │   ├─► Maximize conversion (Fladlien approach)
│   │   │   Strategy: Tight control, conversion-focused
│   │   │   Duration: 10-15 minutes
│   │   │   Method:
│   │   │   • Plant questions if needed
│   │   │   • Use Objection Reversal System
│   │   │   • Turn every Q into reason to buy
│   │   │   • Cut off when energy drops
│   │   │
│   │   └─► Build community (Cage approach)
│   │       Strategy: Open, comprehensive, relationship-building
│   │       Duration: 20-30 minutes
│   │       Method:
│   │       • Answer thoroughly
│   │       • Use to reinforce teaching
│   │       • Build safety and trust
│   │       • Show expertise depth
│   │
│   └─► Check: Audience size
│       │
│       ├─► Small audience (<100)
│       │   └─► Can do more open Q&A (Cage)
│       │       Benefit: Personal connection
│       │
│       └─► Large audience (100+)
│           └─► Need tighter control (Fladlien)
│               Risk: Questions go off-track, energy dissipates
│
├─► Automated evergreen
│   ├─► Strategy: Pre-recorded FAQ (Fladlien efficiency)
│   │   Duration: 10-15 minutes max
│   │   Method:
│   │   • Identify 5-7 most common questions
│   │   • Record answers that advance close
│   │   • Edit tightly
│   │   • Position as "Here's what people always ask..."
│   │
│   └─► Alternative: No Q&A section
│       Method:
│       • Pre-handle all objections in content (Cage)
│       • Offer email/chat support for questions
│       • Send FAQ document post-webinar
│       Benefit: Cleaner flow, no energy drop
│
└─► Hybrid (live with replay sales)
    ├─► Strategy: Balanced approach
    │   Duration: 15-20 minutes
    │   Method:
    │   • Live: Open Q&A (Cage) for attendees
    │   • Edit replay: Cut to best questions (Fladlien)
    │   • Best of both: Community + efficiency
    │
    └─► Tip: Call out great questions during live
        "That's a great question - everyone asks this"
        Result: Validates question for replay viewers
```

## Meta Decision Tree: When Experts Completely Conflict

```
Conflict Question: Experts disagree and context isn't clear. What do I do?

START: Is there empirical data on this?
│
├─► YES - Data exists
│   └─► Default to data
│       • What do the numbers say?
│       • Which expert's approach has better documented results?
│       • Apply that approach
│       • Note: Document which you chose and why
│
├─► NO - No clear data
│   ├─► Check: Can I test both?
│   │   │
│   │   ├─► YES - Testing is viable
│   │   │   └─► Test protocol:
│   │   │       1. Create version A (Expert 1 approach)
│   │   │       2. Create version B (Expert 2 approach)
│   │   │       3. Run both with similar audiences
│   │   │       4. Measure: Registration, attendance, conversion
│   │   │       5. Document results
│   │   │       6. Update synthesis map with findings
│   │   │
│   │   └─► NO - Testing not viable (one shot, can't split)
│   │       └─► Default hierarchy:
│   │           1. Check price point (higher price = Cage bias)
│   │           2. Check sophistication (higher soph = Cage bias)
│   │           3. Check product type (transformation = Fladlien bias)
│   │           4. If still tied: Default to Cage (conservative choice)
│   │
│   └─► Check: Is there a synthesis option?
│       │
│       ├─► YES - Can combine approaches
│       │   └─► Synthesis path:
│       │       • Use Expert 1 for phase A
│       │       • Use Expert 2 for phase B
│       │       • Document synthesis logic
│       │       Example: Cage open, Fladlien close
│       │
│       └─► NO - Truly contradictory
│           └─► Choose based on:
│               • Your strengths (what feels natural?)
│               • Audience preference (what would they respond to?)
│               • Risk tolerance (which is safer?)
│               • Document: "Chose X because Y, will test Z next"
│
└─► UNSURE - Don't know if data exists
    └─► Research protocol:
        1. Check expert's documented case studies
        2. Search for third-party validations
        3. Ask in expert communities
        4. If no data found: Return to "NO" branch above
```

## Using These Decision Trees

### Step-by-Step Process

1. **Identify Your Context**
   - Price point
   - Market sophistication
   - Webinar type
   - Primary goal
   - Product type
   - Audience familiarity

2. **Navigate the Relevant Tree**
   - Start at the root
   - Answer each question honestly
   - Follow the branch that matches your context
   - Land on a recommendation

3. **Apply the Recommendation**
   - Note which expert's approach you're using
   - Understand why it fits your context
   - Implement the specific frameworks indicated

4. **Document Your Choice**
   - Record which path you took
   - Note the context variables
   - Track results
   - Feed learnings back into system

### When Trees Don't Cover Your Situation

If your context doesn't fit these trees:

1. **Identify what's unique** about your situation
2. **Find closest match** in the trees
3. **Adapt the recommendation** to your context
4. **Document the adaptation** for future reference
5. **Consider** whether a new branch should be added

### Quality Check

Before finalizing any decision:

- ✓ Did I consider all relevant context variables?
- ✓ Do I understand why this expert's approach fits?
- ✓ Have I acknowledged conflicts if present?
- ✓ Do I have a testing plan if uncertain?
- ✓ Am I documenting this for future learning?

---

*These decision trees evolve as we gather more data. Treat them as frameworks, not rigid rules. Context and testing always trump theory.*
