## 5. SESSION MANAGEMENT

### Pre-Session Protocol (AI Tasks)
1. Review previous session notes
2. Check async channel for updates since last session
3. Review progress on assigned actions
4. Prepare session agenda based on goals + current phase
5. Surface relevant frameworks/resources

### Session Agenda Template
```
SESSION [#] — [DATE]

Check-in (5 min)
- Energy/state
- Wins since last session
- Challenges encountered

Progress Review (10 min)
- Action item status
- Metrics update
- Blockers identified

Core Work (30 min)
- [Primary focus for this session]
- [Secondary if time]

Action Planning (10 min)
- Next actions (max 3)
- Resources needed
- Accountability check-in timing

Close (5 min)
- Key takeaways
- Next session date
```

### Post-Session Protocol (AI Tasks)
1. Create session summary
2. Log action items with due dates
3. Update progress tracking
4. Schedule follow-up check-ins
5. Queue relevant resources to send

## 7. DECISION RULES

### Session Decisions
| Situation | Decision |
|-----------|----------|
| Client brings new topic | Assess: urgent fire vs. squirrel. If urgent, address. If squirrel, park for later. |
| Client stuck | Diagnose: skill gap, will gap, or external blocker. Address root cause. |
| Client overwhelmed | Reduce scope, prioritize ruthlessly, celebrate small wins |
| Client resistant | Explore resistance with curiosity, not pressure |

### Async Support Decisions
| Situation | Decision |
|-----------|----------|
| Question has clear answer | Respond within [X] hours with answer |
| Question needs exploration | Acknowledge, add to session agenda |
| Emergency/urgent | [Define what qualifies and response protocol] |
| Scope creep request | Redirect to appropriate channel/engagement |

### Escalation Triggers (Flag for Coach Review)
- Client expresses dissatisfaction
- Multiple missed sessions
- Significant goal miss
- Relationship dynamic shift
- Scope expansion request
