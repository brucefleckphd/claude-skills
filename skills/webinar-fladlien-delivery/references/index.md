## Supporting Frameworks (Jack/10)

### G1. Show-and-Tell Principle (Jack)
**File:** `framework-show-and-tell-principle-pinksheet.md`
**Core Focus:** "Show actual leaderboard, not slide with numbers" (Real > Polished)
**When to Use:** Adding authenticity throughout webinar
**Applies To:** Authority, testimonials, product demos, guarantee

### G2. Slide Iteration Process (10)
**File:** `framework-slide-iteration-process-pinksheet.md`
**Core Focus:** Words → Speaker Notes → Visuals only (one slide becomes seven)
**When to Use:** Creating or polishing slide decks
**Applies To:** All sections (visual slides, not text-heavy)

### G3. Defocus Animation (10)
**File:** `framework-defocus-animation-pinksheet.md`
**Core Focus:** Blur everything except current focus point
**When to Use:** Showing progress through multi-step processes
**Applies To:** Roadmap Reveal, Close Sequence, Bonus Compounding

### G4. 25% Content Cut Rule (10)
**File:** `framework-25-percent-cut-rule-pinksheet.md`
**Core Focus:** Cut 25% after first pass (easier to subtract than add)
**When to Use:** Final content editing before going live
**Applies To:** All sections (honors Value Paradox)

## The Polish Workflow

**After all content is created:**

1. **Cut First** (G4)
   - Review full webinar
   - Identify 25% to remove
   - Cut redundancy, tangents, fluff

2. **Visualize Second** (G2)
   - Move text to speaker notes
   - Replace with images/diagrams
   - One slide → Seven visual slides

3. **Authenticate Third** (G1)
   - Replace polished with real
   - Add actual screenshots
   - Use real testimonials/emails

4. **Navigate Fourth** (G3)
   - Add progress indicators
   - Implement defocus animations
   - Make navigation clear

5. **Final Review**
   - Check all four principles applied
   - Validate against other frameworks
   - Ready to ship

- [ ] G1: Is product demo the real interface vs mockup?

- [ ] G3: Do I show progress through roadmap?
- [ ] G3: Do I highlight current focus in sequences?
